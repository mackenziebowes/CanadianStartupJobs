# Agent Tasks 003-1

## Context
We are building an Admin UI to manage VCs, Companies, and Jobs. Instead of creating 12 separate page files (Create/Read/Update/Delete for 3 resources), we will use a **Configuration-Driven** approach. We need a central "Source of Truth" that defines how each resource looks and behaves.

## Goal
Step 1: Create the centralized configuration file that defines the schema, labels, and API endpoints for our three main resources (VCs, Companies, Jobs).

### Task Details

**File:** `frontend/admin/config/admin-resources.ts` (Create this file)

**Imports to add:**
- None (this is a pure JS/TS object).

**Steps:**
1.  Export a constant named `RESOURCES`.
2.  Define the `vcs` key with:
    - `label`: "Venture Capitals"
    - `endpoint`: "/api/vcs"
    - `fields`: Array including `name` (text), `portfolio_url` (url), and `status` (badge/text).
3.  Define the `companies` key with:
    - `label`: "Companies"
    - `endpoint`: "/api/companies"
    - `fields`: Array including `name` (text), `vc_parent_id` (relationship/text), and `careers_url` (url).
4.  Define the `jobs` key with:
    - `label`: "Jobs"
    - `endpoint`: "/api/jobs"
    - `fields`: Array including `title` (text), `company_id` (relationship/text), `salary` (text), and `description` (textarea - hide in list view).
5.  *Note:* Ensure every field object has keys for `name`, `label`, `type`, `list` (boolean: show in table?), and `edit` (boolean: show in form?).

# Agent Tasks 003-2

## Context
We have our configuration map (003-1). Now we need a **reusable Table component** that can render *any* of those resources. This component will replace the need for separate `VCTable`, `CompanyTable`, and `JobTable` components.

## Goal
Step 2: Build a `GenericResourceTable` component that accepts a `resourceKey` prop and dynamically renders the table headers based on the config.

### Task Details

**File:** `apps/web/components/admin/generic-resource-table.tsx`

**Imports to add:**
- `RESOURCES` from `@/config/admin-resources`.
- Table primitives from `@/components/ui/table`:
  - `Table`, `TableHeader`, `TableRow`, `TableHead`, `TableBody`, `TableCell`.

**Steps:**
1.  Create a component `GenericResourceTable` that accepts props: `{ resourceKey, data }`.
    - *Note:* We are assuming `data` is passed in for now (dumb component).
2.  Inside the component, look up the config: `const config = RESOURCES[resourceKey];`.
3.  Render the `<TableHeader>`:
    - Map over `config.fields`.
    - If `field.list === true`, render a `<TableHead>{field.label}</TableHead>`.
    - Add one final empty `<TableHead>` for actions (Edit/Delete buttons).
4.  Render the `<TableBody>`:
    - Map over the `data` array to create rows.
    - Inside each row, map over `config.fields` again.
    - If `field.list === true`, render `<TableCell>{row[field.name]}</TableCell>`.
5.  Add a placeholder "Edit" button in the final cell of each row.

# Agent Tasks 003-3

## Context
We have a dumb table (003-2). Now we need a **Page View** that connects the URL to the data. When a user visits `/admin/vcs`, it should know to fetch VCs and render the Generic Table.

## Goal
Step 3: Create a dynamic Admin List Page that uses the URL parameter to determine which data to fetch and display.

### Task Details

**File:** `apps/web/app/admin/[resource]/page.tsx` (Next.js App Router structure)

**Imports to add:**
- `RESOURCES` from `@/config/admin-resources`.
- `GenericResourceTable` from `@/components/admin/generic-resource-table`.
- `notFound` from `next/navigation` (for handling invalid URLs).

**Steps:**
1.  Create a page component that accepts `params` (specifically `params.resource`).
2.  Validate the resource:
    - Check if `RESOURCES[params.resource]` exists.
    - If not, call `notFound()`.
3.  Fetch Data (Client-side for now, or Server Component):
    - *Assumption:* Use a simple `fetch` to `RESOURCES[params.resource].endpoint`.
    - Store data in a variable.
4.  Render the Page:
    - Display a title: `<h1>{config.label}</h1>`.
    - Render `<GenericResourceTable resourceKey={params.resource} data={data} />`.
5.  Add a "Create New" button above the table that links to `/admin/${params.resource}/new`.

# Agent Tasks 003-4

## Context
We can list data. Now we need to **Create/Edit** data. Instead of hardcoding forms, we need a **Generic Form** that reads the config and generates the correct inputs (Text inputs for names, Textareas for descriptions).

## Goal
Step 4: Build a `GenericResourceForm` component that dynamically renders inputs based on the resource configuration.

### Task Details

**File:** `apps/web/components/admin/generic-resource-form.tsx`

**Imports to add:**
- `RESOURCES` from `@/config/admin-resources`.
- `Input` from `@/components/ui/input`.
- `Textarea` from `@/components/ui/textarea`.
- `Label` from `@/components/ui/label`.
- `Button` from `@/components/ui/button`.
- `useState` or `useForm` (react-hook-form recommended, but let's stick to simple state for this step).

**Steps:**
1.  Create component `GenericResourceForm` accepting props: `{ resourceKey, initialData, onSubmit }`.
2.  Initialize state `formData` with `initialData` (or empty object).
3.  Look up `config = RESOURCES[resourceKey]`.
4.  Render the form fields:
    - Map over `config.fields`.
    - **Filter:** Only show fields where `field.edit === true`.
    - **Switch:** Check `field.type`.
      - Case `'text'`: Render `<Label>` + `<Input>`.
      - Case `'url'`: Render `<Label>` + `<Input type="url">`.
      - Case `'textarea'`: Render `<Label>` + `<Textarea>`.
    - Bind the `value` and `onChange` to `formData`.
5.  Render a "Save" button at the bottom.
6.  On form submit, prevent default and call `onSubmit(formData)`.
